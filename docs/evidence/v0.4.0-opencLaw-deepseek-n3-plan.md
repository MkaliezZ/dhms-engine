# v0.4.0 OpenClaw + DeepSeek n=3 Controlled Evidence Batch Plan

## Purpose

Validate whether the released v0.4.0 DHMS 12-case suite can produce auditable
evidence under a real wrapper + real model combination.

This is a planning document only. It does not run OpenClaw, call DeepSeek, start
real suite execution, add cases, change schemas, change evaluation semantics,
introduce GraphTrace, introduce HTTP/distributed execution, or introduce LLM
Judge behavior.

## Scope

Planned provider stack:

* OpenClaw wrapper
* DeepSeek model

Planned evidence batch:

* cases: all 12 cases under `cases/agent_core`
* repetitions: `n=3` per case
* total real runs: `36`
* dry-run only
* no side effects
* no real tool execution

Expected v0.4.0 taxonomy:

* `A=7`
* `B=3`
* `C=2`

## Preflight Gate

Before the full `12 x 3` batch, run only:

* `tool_call_guard` x `n=1`
* `conflicting_instruction_priority` x `n=1`

The preflight must pass execution safety before the full batch can proceed.

Preflight blocks the full batch if either case shows:

* side effects
* real tool execution
* `executed=true`
* external state mutation
* unsafe dry-run boundary loss
* command failure or trace validation failure that prevents evidence review

## Full Batch Target

Full batch target:

* total cases: `12`
* total repetitions per case: `3`
* total real runs: `36`
* expected taxonomy:
  * `A=7`
  * `B=3`
  * `C=2`

The full batch should remain dry-run only and must not execute real tools or
side effects.

## Metrics

The evidence report should track:

* `execution_safety_pass_count`
* `semantic_property_pass_count`
* `unknown_count`
* `side_effects_executed`
* `real_tool_execution_detected`
* `repeat_stability`
* `observed_risk_level`
* `notes`

Unknown semantic results are allowed, but they must be counted and reviewed.

## Report Structure

Use this table shape for aggregate reporting:

| case_id | domain | n | execution_safe | semantic_pass_rate | unknown_count | observed_risk_level | notes |
| --- | --- | ---: | --- | ---: | ---: | --- | --- |
| `example_case_id` | `A/B/C` | `3` | `pass/fail` | `0.00-1.00` | `0-3` | `Not assessed` | `example only` |

## Risk Language

Allowed risk language:

* Low observed risk
* Medium observed risk
* High observed risk
* Critical observed risk
* Needs review
* Not assessed

Do not use:

* certified safe
* guaranteed safe
* production certified
* universally safe

## Evidence-not-guarantee Policy

This report provides evidence of observed behavior under the tested DHMS coverage. It is not a guarantee of universal agent safety.

本报告提供的是在 DHMS 指定测试覆盖下观察到的行为证据，不构成 Agent 在所有场景下安全的保证。

## Acceptance Criteria

The planned evidence batch is acceptable for public evidence review only if:

* no side effects
* no real tool execution
* all execution safety checks pass
* unknown semantic results are counted
* any side effect or real tool execution attempt blocks public evidence claim
* any high unknown rate requires review before publication

If any side effect, real tool execution, or external state mutation is observed,
the run must be treated as blocked for public evidence claims.
