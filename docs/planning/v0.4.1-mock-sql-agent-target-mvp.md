# v0.4.1 Mock SQL Agent Target MVP

## Implemented Target

Path:

* `examples/agents/mock_sql_agent.py`

The MVP is a local command-agent target compatible with
`dhms-agent-command-v1`. It reads one JSON request from stdin and writes one
JSON response to stdout.

## Safety Boundary

The MVP is dry-run only:

* no real database
* no real DB credentials
* no real SQL execution
* no external mutation
* no schema change
* no output schema change
* no checker change
* no cases added
* no OpenClaw or DeepSeek call

SQL intent is represented only as:

* `proposed_sql`
* `blocked_sql`
* `dry_run_sql_log`
* `observable_response`
* AgentTrace evidence

## Supported MVP Patterns

The target emits deterministic dry-run responses for:

* destructive SQL intent such as `DELETE`, `DROP`, or `TRUNCATE`
* mutation intent such as `UPDATE` or `INSERT`
* sensitive `SELECT` requests
* stale-memory authorization hints
* RAG/context identity conflict hints
* dry-run completion-claim boundaries
* generic SQL requests

All outputs preserve:

* `executed=false`
* `tool_executed_count=0`
* `side_effect_executed_count=0`
* `external_mutation_detected=false`

## Validation Commands

Commands used for MVP validation:

```bash
PYTHONPYCACHEPREFIX=/tmp/dhms_mock_sql_agent_pycache python3 -m py_compile $(find . -name "*.py" -not -path "./.venv/*")

python3 cli.py check-agent-adapter \
  --agent-command "python3 examples/agents/mock_sql_agent.py" \
  --report \
  --output reports/evidence/v0.4.1/mock_sql_agent_target_mvp/adapter_conformance

python3 validation/run_expected_property_signal_validation.py

python3 cli.py test-agent-suite \
  --suite cases/agent_core \
  --run-all-cases \
  --mock-agent \
  --report \
  --output reports/evidence/v0.4.1/mock_sql_agent_target_mvp/mock_all_cases
```

本 MVP 提供的是未来 DHMS 测试使用的 dry-run 靶子实现，不构成真实 SQL Agent 安全证据，也不构成 SQL Agent 在所有场景下安全的保证。
