# DHMS Provider API Matrix

This matrix records adapter assumptions for DHMS Provider Expansion v1. It is implementation guidance only; DHMS Spec, Contract, Binding, and metric semantics remain unchanged.

| Provider | API style | Endpoint/base URL | Auth header | Request shape | Response extraction | Env vars | Default model strategy | Alias strategy | docs_verified_live | Conformance status | Live smoke status | Caveats |
|---|---|---|---|---|---|---|---|---|---|---|---|---|
| deepseek | OpenAI-compatible Chat Completions | `https://api.deepseek.com/chat/completions` | `Authorization: Bearer` | `model`, `messages`, `temperature=0` | `choices[0].message.content` | `DEEPSEEK_API_KEY`, `DEEPSEEK_MODEL`, `DEEPSEEK_FLASH_MODEL`, `DEEPSEEK_PRO_MODEL` | `DEEPSEEK_MODEL` or alias default | `flash`, `pro`, direct model ID | true | static conformance covered | run via provider smoke | DeepSeek docs currently list `deepseek-v4-flash` and `deepseek-v4-pro`; env override preferred. |
| openai | Responses API | `https://api.openai.com/v1/responses` | `Authorization: Bearer` | `model`, `input`, `temperature=0` | `output_text`, fallback traversal of `output[].content[].text` | `OPENAI_API_KEY`, `OPENAI_MODEL`, `OPENAI_MINI_MODEL` | `OPENAI_MODEL` or direct model ID | `latest`, `mini`, direct model ID | true | static conformance covered | skipped unless key present | Current model IDs may change; direct passthrough and env override are preferred. |
| anthropic | Messages API | `https://api.anthropic.com/v1/messages` | `x-api-key`, `anthropic-version` | `model`, `max_tokens`, `temperature=0`, `messages` | `content[0].text` traversal | `ANTHROPIC_API_KEY`, `ANTHROPIC_MODEL`, `ANTHROPIC_VERSION`, `ANTHROPIC_SONNET_MODEL`, `ANTHROPIC_OPUS_MODEL`, `ANTHROPIC_HAIKU_MODEL` | `ANTHROPIC_MODEL` or alias default | `sonnet`, `opus`, `haiku`, direct model ID | true | static conformance covered | skipped unless key present | Anthropic model names are versioned; env override preferred. |
| qwen | OpenAI-compatible Chat Completions | `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions` | `Authorization: Bearer` | `model`, `messages`, `temperature=0` | `choices[0].message.content` | `DASHSCOPE_API_KEY`, `QWEN_MODEL`, `QWEN_BASE_URL`, `QWEN_REGION`, `QWEN_PLUS_MODEL`, `QWEN_TURBO_MODEL` | `QWEN_MODEL` or `qwen-plus` | `plus`, `turbo`, direct model ID | true | static conformance covered | skipped unless key present | Aliyun doc page was live-opened but sparse in fetched text; adapter uses DashScope OpenAI-compatible convention. |
| kimi | OpenAI-compatible Chat Completions | `https://api.moonshot.ai/v1/chat/completions` | `Authorization: Bearer` | `model`, `messages`, `temperature=0` | `choices[0].message.content` | `MOONSHOT_API_KEY`, `KIMI_API_KEY`, `KIMI_MODEL`, `KIMI_BASE_URL`, `KIMI_K2_MODEL` | `KIMI_MODEL` or Kimi K2 default | `k2`, direct model ID | true | static conformance covered | skipped unless key present | Moonshot/Kimi docs live-opened; model IDs evolve, env override preferred. |
| gemini | Native generateContent | `https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent` | API key query parameter | `contents[].parts[].text`, `generationConfig.temperature=0` | `candidates[0].content.parts[0].text` | `GEMINI_API_KEY`, `GEMINI_MODEL`, `GEMINI_FLASH_MODEL`, `GEMINI_PRO_MODEL` | `GEMINI_MODEL` or flash default | `flash`, `pro`, direct model ID | true | static conformance covered | skipped unless key present | Gemini model names change; direct passthrough and env override preferred. |
| mistral | Chat Completions | `https://api.mistral.ai/v1/chat/completions` | `Authorization: Bearer` | `model`, `messages`, `temperature=0` | `choices[0].message.content`, including string/list chunks | `MISTRAL_API_KEY`, `MISTRAL_MODEL`, `MISTRAL_SMALL_MODEL`, `MISTRAL_MEDIUM_MODEL`, `MISTRAL_LARGE_MODEL` | `MISTRAL_MODEL` or small default | `small`, `medium`, `large`, direct model ID | true | static conformance covered | skipped unless key present | Mistral content may be string or structured list; parser handles both. |
| fallback | Local deterministic fallback | local | none | local deterministic text | local string | none | local fallback | `fallback_mock -> fallback` | false | static conformance covered | no live smoke | Used only as explicit fallback/unavailable marker, not real validation. |
