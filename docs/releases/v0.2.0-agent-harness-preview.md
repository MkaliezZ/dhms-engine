# v0.2.0-agent-harness-preview

DHMS Agent Harness v1 MVP Preview

DHMS Agent Harness v1 MVP Preview introduces a dry-run, SDK-free, local BYOA command-agent testing harness with trace diagnosis, suite reports, bad-agent demos, and JSON / Markdown / static HTML outputs.

## What this preview is

Agent Harness is a local, dry-run-first harness for testing user-owned agents. It currently supports mock agents and local command agents.

Users do not need to embed a DHMS SDK into their core agent code. Integration is wrapper/protocol based through `dhms-agent-command-v1`: the user-owned wrapper receives a JSON request on stdin and returns an AgentTrace-compatible JSON object on stdout.

SDK-free by default, wrapper-based integration, non-invasive to agent core logic.

DHMS treats returned tool calls and side effects as trace evidence only. DHMS does not grant permission to execute tools, does not call user tools, and does not certify production agent safety.

## Supported commands

Mock single-case:

```bash
python3 cli.py test-agent \
  --mock-agent \
  --input "Check the refund policy and issue a refund if eligible." \
  --n 1 \
  --report \
  --output reports/agent_harness_preview/mock_single
```

Command single-case:

```bash
python3 cli.py test-agent \
  --agent-command "python3 examples/agents/sample_json_agent.py" \
  --input "Check the refund policy and issue a refund if eligible." \
  --n 1 \
  --report \
  --output reports/agent_harness_preview/command_single
```

Mock suite:

```bash
python3 cli.py test-agent-suite \
  --suite cases/agent_core \
  --mock-agent \
  --n 1 \
  --report \
  --output reports/agent_harness_preview/mock_suite
```

Command suite:

```bash
python3 cli.py test-agent-suite \
  --suite cases/agent_core \
  --agent-command "python3 examples/agents/sample_json_agent.py" \
  --n 1 \
  --report \
  --output reports/agent_harness_preview/command_suite
```

## Report outputs

Single-case runs generate:

* `agent_harness_report.json`
* `agent_harness_report.md`
* `agent_harness_report.html`

Suite runs generate:

* `suite_agent_report.json`
* `suite_agent_report.md`
* `suite_agent_report.html`

JSON is for machine reading / integration. Markdown is for GitHub review. Static HTML is for demos, screenshots, and non-technical inspection.

## Diagnosis highlights

This preview includes diagnosis labels for:

* `side_effect_risk`
* `side_effect_guard_passed`
* `dry_run_policy_violation`
* `unsafe_side_effect_execution`
* `command_adapter_invalid_json`
* `command_adapter_wrong_protocol`
* `command_adapter_timeout`
* `command_adapter_trace_validation_error`
* `command_adapter_nonzero_exit`
* `expected_agent_property_violation`
* `insufficient_trials`

Invalid JSON and wrong protocol outputs are reported as command-adapter failures. Executed side effects are Critical. `n=1` produces preliminary results and should not be treated as a general reliability claim.

## Bad-agent demos

The preview includes intentionally broken local agents for safe failure demonstrations:

* `examples/agents/bad_invalid_json_agent.py`
* `examples/agents/bad_wrong_protocol_agent.py`
* `examples/agents/bad_dry_run_false_agent.py`
* `examples/agents/bad_executed_side_effect_agent.py`

These examples demonstrate safe handling of invalid JSON, wrong protocol version, dry-run policy violation, and unsafe side-effect execution evidence.

## Safety model

* dry-run only
* local BYOA command agents only
* returned side effects are trace evidence only
* DHMS does not execute user tool calls
* DHMS does not certify production agent safety
* user wrappers should not connect to production tools during testing

## What is not included

* HTTP adapter
* remote agent adapter
* real tool execution permission
* production certification
* SaaS/dashboard/server
* SDK instrumentation requirement

## Caveats

* MVP preview, not production certification
* `n=1` is preliminary
* sample agents are not production agents
* local command process is user-owned
* DHMS cannot prevent a malicious user-owned process from doing things internally; users should run dry-run/sandbox wrappers
