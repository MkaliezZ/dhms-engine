# Reproduce v0.3.1 Mock/Local Multi-case Report

## Purpose

This guide reproduces the DHMS Agent Harness v0.3.1 multi-case mock/local report
from a fresh local checkout. It is intended for external developers who want to
verify the standardized `execution_summary.json` schema and readable multi-case
Markdown report without running a real agent provider.

This reproduction path is:

* mock/local only
* dry-run only
* no OpenClaw required
* no DeepSeek required
* no provider API key required
* no real tool execution
* no side effects

## Command

Run from the repository root:

```bash
python3 cli.py test-agent-suite \
  --suite cases/agent_core \
  --run-all-cases \
  --mock-agent \
  --report \
  --output reports/reproducibility/v0.3.1_mock_all_cases
```

## Expected Outputs

The command should write:

* `reports/reproducibility/v0.3.1_mock_all_cases/execution_summary.json`
* `reports/reproducibility/v0.3.1_mock_all_cases/suite_agent_report.md`
* `reports/reproducibility/v0.3.1_mock_all_cases/suite_agent_report.json`
* `reports/reproducibility/v0.3.1_mock_all_cases/suite_agent_report.html`
* per-case reports under `reports/reproducibility/v0.3.1_mock_all_cases/per_case/`

The committed lightweight reference artifacts are:

* `docs/reproducibility/artifacts/v0.3.1_mock_all_cases/execution_summary.json`
* `docs/reproducibility/artifacts/v0.3.1_mock_all_cases/suite_agent_report.md`

## Expected Summary

The mock/local multi-case run should report:

* `total_cases=6`
* `taxonomy_summary` counts:
  * `A=5`
  * `B=1`
  * `C=0`
* `execution_summary.json` exists
* `suite_agent_report.md` exists
* no real tool execution
* no side effects

The `execution_summary.json` top-level schema should include:

* `schema_version`
* `run_metadata`
* `suite_summary`
* `taxonomy_summary`
* `consistency_summary`
* `cases`

Each case entry should include:

* `case_id`
* `taxonomy_domain`
* `taxonomy_label`
* `execution_safety_result`
* `semantic_property_result`
* `final_status`

## Scope

This is a reproducibility package for v0.3.1 mock/local reporting only. It does
not add new real OpenClaw or DeepSeek evidence, does not use a real LLM Judge,
and does not claim production certification, multi-model certification,
system-level sandbox proof, or full-suite production validation.
