# DHMS Local Command Proposal Examples v0

These examples are inert documentation/data only.

They are not shell tutorials, command execution instructions, terminal usage
guides, or operational recipes. They must not be copied into a terminal.

Local command proposals are treated as data. DHMS observes the proposal,
normalizes the intent, classifies the risk, assigns a policy decision, and
records trace evidence before any execution boundary.

For v1.1.3, allowed policy decisions remain:

* `HOLD`
* `BLOCK`
* `FAIL_CLOSED`

`RELEASE` is not used in v1.1.3.

Future controlled proof work must remain synthetic, deterministic,
constrained, non-production, and explicitly approved before implementation.

The examples in `inert_examples.json` are compact reader aids linked to the
static local command proposal manifest:

`benchmarks/dhms_local_command_proposals_v0/cases.json`

They do not execute command strings, argv arrays, shells, subprocesses,
terminal sessions, command runners, MCP tools, E2B tools, Codex, Claude,
OpenClaw, DeepSeek, provider SDKs, or agent SDKs.
